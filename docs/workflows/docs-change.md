# 文档变更 Workflow

适用：仅修改/新增文档、目录索引、README，**不改代码或配置**。

## 开始前的分析方法
- 确认变更范围：新增/调整的文档类型与对应 BRN/XRD 编号。
- 找到“对的位置”：查看 `docs/index.md`、父级 README，避免重复或错位。
- 梳理上下游引用：哪些索引/导航需要同步更新。

## 必须阅读的文档
- `AGENTS.md`
- `docs/README.md`、`docs/index.md`
- 目标子目录的 README（如 `docs/specs/README.md`）
- 对应 BRN/XRD 文档（确保口径一致）

## 流程 checklist
- [ ] 明确变更目的与目标读者。
- [ ] 梳理需更新的索引/导航链路。
- [ ] 编辑目标文档（保持原有结构与编号规则）。
- [ ] 同步更新相关 README/索引（含上层目录）。
- [ ] 自检格式：标题层级、表格、链接路径。
- [ ] 标记 todowrite/迭代记录（若任务关联 BRN）。

## 流程的强制约束
- 不新增未经定义的编号；遵循 BRN→XRD 继承规则。
- 不删除用户提供的原始决策材料；去重时保留 SSOT。
- 遵守“先读后写”，不越权修改 `origin/`。
- 若涉及自动产物，放入 `x-` 开头目录。

## 流程的优化目标
- 最小扰动：改动集中、路径清晰。
- 提升可导航性：交叉链接指向明确的“下一步”。
- 复用已有段落/表格，减少重复表述。

## 质量标准
- 链接可点击且路径正确（相对路径优先）。
- 结构与语气与现有文档一致；ASCII 优先。
- 关键信息（责任、状态、口径）有明确位置。

## 后续动作
- 在相关 todowrite/phrase 记录完成情况或待办。
- 若文档指导流程发生变化，通知涉及流程的维护人。
